## 3-1. write.js — `errorData` 참조 에러 (중요도: 높음)

### 문제

`src/components/board/write.js` 236번째 줄에서 `errorData`라는 변수를 사용하는데, 이 변수는 코드 어디에도 선언되어 있지 않아요. 게시글 수정이 서버에서 실패하면 `ReferenceError`가 발생해요.

### 해결 방법

**Step 1.** `errorData.message`를 문자열로 교체하세요.

```js
// 수정 전 (src/components/board/write.js 236번째 줄)
if (!editRes.ok) {
  throw new Error(errorData.message || '수정 실패');
}

// 수정 후
if (!editRes.ok) {
  throw new Error('수정 실패');
}
```

만약 서버에서 보내주는 에러 메시지를 사용자에게 보여주고 싶다면:

```js
// 수정 후 (상세 버전)
if (!editRes.ok) {
  const errorData = await editRes.json().catch(() => ({}));
  throw new Error(errorData.message || '수정 실패');
}
```

`await editRes.json().catch(() => ({}))`가 하는 일을 단계별로 설명하면:
1. `editRes.json()` — 서버 응답을 JSON으로 변환 시도
2. `.catch(() => ({}))` — JSON 변환이 실패하면(서버가 JSON을 안 보냈으면) 빈 객체 `{}`를 사용
3. 이렇게 하면 어떤 상황에서도 에러가 나지 않아요!

> **쉽게 말하면:** 선물 상자를 열었는데 빈 상자일 수 있잖아요? 그래서 "상자가 비었으면 빈 봉투라도 준비해둬"라고 하는 거예요.

## 3-2. boardDetailUI.js — post.title과 comment.content XSS 누락 (중요도: 높음)

### 문제

`src/components/board/boardDetailUI.js`에서 `post.content`에는 `escapeHtml`을 적용했지만, `post.title`(120번째 줄)과 `comment.content`(50번째 줄)에는 적용하지 않았어요.

### 해결 방법

**Step 1.** `BoardDetailContent` 함수에서 `post.title`을 수정하세요.

```js
// 수정 전 (src/components/board/boardDetailUI.js 119~121번째 줄)
<h1 class="text-3xl font-bold text-[#1a3a35] leading-tight flex-1">
  ${post.title}

// 수정 후
<h1 class="text-3xl font-bold text-[#1a3a35] leading-tight flex-1">
  ${escapeHtml(post.title)}
</h1>
```

**Step 2.** `Comment` 함수에서 `comment.content`를 수정하세요.

```js
// 수정 전 (src/components/board/boardDetailUI.js 50번째 줄)
<p id="comment-content-${comment.commentId}" class="text-gray-600 text-[14px] text-left leading-relaxed whitespace-pre-wrap">${comment.content}</p>

// 수정 후
<p id="comment-content-${comment.commentId}" class="text-gray-600 text-[14px] text-left leading-relaxed whitespace-pre-wrap">${escapeHtml(comment.content)}</p>
```

**이미 import가 되어 있으니** 파일 상단의 `import { escapeHtml } from '../../utils/escapeHtml.js';`는 그대로 두면 돼요.

**규칙을 기억하세요:** 사용자가 입력할 수 있는 데이터(제목, 내용, 닉네임, 댓글 등)는 **무조건 `escapeHtml`로 감싸기!** 반면 시스템이 만든 데이터(카테고리 코드, ID 숫자, 날짜 등)는 안 해도 괜찮아요.

> **쉽게 말하면:** 택배 상자를 열 때 커터칼이 들어있을 수 있으니 장갑을 끼는 거예요. 사용자 입력은 "외부에서 온 택배"라서 항상 조심해야 해요!

## 3-3. write.js — 모듈 최상위 token 변수 (중요도: 중간)

### 문제

`src/components/board/write.js` 17번째 줄에 모듈 최상위 `const token`이 있어서, SPA에서 토큰이 바뀌어도 이미지 업로드 시 옛 토큰이 사용돼요.

### 해결 방법

**Step 1.** 17번째 줄의 모듈 레벨 token 선언을 **삭제**하세요.

```js
// 수정 전 (src/components/board/write.js 17번째 줄)
const token = localStorage.getItem('token');

// 수정 후 — 이 줄을 삭제!
```

**Step 2.** 265번째 줄의 이미지 업로드 이벤트 핸들러 안에서 token을 직접 가져오세요.

```js
// 수정 전 (src/components/board/write.js 265번째 줄 근처)
document.getElementById('write-image').addEventListener('change', async (e) => {
  const file = e.target.files[0];
  if (!file) {
    return;
  }
  const formData = new FormData();
  formData.append('file', file);
  const uploadRes = await fetch(`${BASE_URL}/images`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },  // ← 오래된 모듈 레벨 token
    body: formData,
  });

// 수정 후
document.getElementById('write-image').addEventListener('change', async (e) => {
  const file = e.target.files[0];
  if (!file) {
    return;
  }
  const token = localStorage.getItem('token');   // ← 함수 안에서 직접 가져오기!
  const formData = new FormData();
  formData.append('file', file);
  const uploadRes = await fetch(`${BASE_URL}/images`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },
    body: formData,
  });
```

**주의:** 이 파일의 다른 곳(168번째 줄, 20번째 줄)에서는 이미 함수 내부에서 `const token = localStorage.getItem('token')`을 사용하고 있어서 그 부분은 수정할 필요 없어요. 17번째 줄만 삭제하면 돼요!

> **쉽게 말하면:** 아침에 날씨를 확인하고 "오늘은 따뜻하네" 하고 기억해둔 다음, 저녁에 외출할 때 그 기억만 믿으면 안 돼요. 저녁에는 추울 수 있으니까 그때 다시 확인해야 해요!

## 3-5. window 전역 함수 → 이벤트 위임으로 전환 (중요도: 중간)

### 문제

`boardDetail.js`에 6개, `pokedex.js`에 3개의 window 전역 함수가 있어요.

### 해결 방법

이건 여러 파일에 걸친 큰 리팩토링이라서, **가장 영향이 큰 boardDetail.js부터** 순서대로 해결하는 것을 추천해요.

**Step 1.** `boardDetailUI.js`의 HTML에서 `onclick` 대신 `data-` 속성을 사용하세요.

```js
// 수정 전 (boardDetailUI.js — Comment 함수 안)
<button onclick="toggleEditMode(${comment.commentId})">수정</button>
<button onclick="handleDeleteComment(${comment.commentId})">삭제</button>

// 수정 후
<button data-action="edit-comment" data-id="${comment.commentId}">수정</button>
<button data-action="delete-comment" data-id="${comment.commentId}">삭제</button>
```

```js
// 수정 전 (boardDetailUI.js — BoardDetailContent 함수 안)
<button onclick="handleEditPost(${post.postId})">수정</button>
<button onclick="handleDeletePost(${post.postId})">삭제</button>

// 수정 후
<button data-action="edit-post" data-id="${post.postId}">수정</button>
<button data-action="delete-post" data-id="${post.postId}">삭제</button>
```

**Step 2.** `boardDetail.js`에서 window 함수 대신 이벤트 위임을 사용하세요.

```js
// 수정 후 (boardDetail.js — setupCommentEvents 함수 끝부분에 추가)

// 이벤트 위임으로 댓글/게시글 액션 처리
document.addEventListener('click', (e) => {
  const btn = e.target.closest('[data-action]');
  if (!btn) return;

  const action = btn.dataset.action;
  const id = Number(btn.dataset.id);

  switch (action) {
    case 'edit-comment':
      toggleEditMode(id);
      break;
    case 'delete-comment':
      handleDeleteComment(id);
      break;
    case 'edit-post':
      handleEditPost(id);
      break;
    case 'delete-post':
      handleDeletePost(id);
      break;
  }
});
```

**Step 3.** window 전역 함수 선언을 일반 함수로 바꾸세요.

```js
// 수정 전
window.toggleEditMode = (commentId) => { ... };

// 수정 후 — window. 제거!
function toggleEditMode(commentId) { ... }
```

나머지 5개 함수도 `window.` 접두사를 제거하면 돼요.

> **쉽게 말하면:** 회사에서 "김대리!" 하고 부르면 김대리가 여러 명일 때 혼란이 생겨요. 대신 내선번호로 전화하면 정확히 한 사람에게만 연결돼요. `data-action`은 내선번호 같은 역할이에요!

## 3-6. header.js — location.reload() 뒤의 코드 (중요도: 중간)

### 문제

`src/components/header.js` 86~91번째 줄에서 `location.reload()` 후에 `window.location.href = '/'`가 있어서, 의도대로 동작하지 않을 수 있어요.

### 해결 방법

**Step 1.** `location.reload()` 줄을 삭제하고 `window.location.href = '/'`만 남기세요.

```js
// 수정 전 (src/components/header.js 83~92번째 줄)
document.addEventListener('click', (e) => {
  if (e.target.classList.contains('logout-btn')) {
    localStorage.removeItem('token');
    localStorage.removeItem('userId');
    location.reload();

    // 로그아웃 후 홈으로 이동
    window.location.href = '/';
  }
});

// 수정 후
document.addEventListener('click', (e) => {
  if (e.target.classList.contains('logout-btn')) {
    localStorage.removeItem('token');
    localStorage.removeItem('userId');
    window.location.href = '/';    // 홈으로 이동 (자동으로 새로고침됨!)
  }
});
```

`window.location.href = '/'`는 브라우저에게 "홈 URL로 이동해"라고 하는 명령이에요. 이 과정에서 페이지가 완전히 새로 로드되기 때문에 `location.reload()`가 필요 없어요.

> **쉽게 말하면:** "집에 가서 다시 나와" 대신 "집에 가"만 하면 돼요. 집에 가면 자동으로 다시 시작하니까요!

## 3-7. main.js — import 경로 대소문자 불일치 (중요도: 중간)

### 문제

`src/main.js`에서 `./components/auth/login.js`(소문자)로 import하는데, 실제 폴더는 `Auth`(대문자)예요.

### 해결 방법

**방법 A (추천): 폴더 이름을 소문자로 통일**

터미널에서 다음 명령을 실행하세요:

```bash
# Git에서 폴더 이름 변경 (대소문자 변경은 git mv를 써야 안전해요)
cd /Users/toto/devel/edu/pocket-archive
git mv src/components/Auth src/components/auth-temp
git mv src/components/auth-temp src/components/auth
```

왜 두 번 이동하느냐면, macOS에서는 `Auth` → `auth`를 직접 바꾸면 "같은 이름"이라고 판단해서 무시할 수 있어요. 중간에 다른 이름(`auth-temp`)을 거치면 확실하게 바뀌어요.

**방법 B: import 경로를 실제 폴더명과 맞추기**

```js
// 수정 전 (src/main.js 4~5번째 줄)

// 수정 후
import { Login, initLogin } from './components/Auth/login.js';
import { Register, initRegister } from './components/Auth/register.js';
```

> **쉽게 말하면:** 편지를 보낼 때 주소를 "서울시 강남구"라고 쓰든 "서울시 강남區"라고 쓰든 우체부 아저씨(macOS)는 찾아가지만, 기계 배송(Linux)은 정확히 같은 글자만 인정해요!

**Step 1.** `PokemonModalContent` 함수의 리턴 문자열에서 `<script>...</script>` 부분을 삭제하세요.

```js
// 삭제할 부분 (src/components/pokedex/pokedexUI.js 544~566번째 줄)

  <script>
    (function() {
      let idx = 0;
      const TOTAL = 6;

      window.pmSlide = function(dir) {
        idx = Math.max(0, Math.min(idx + dir, TOTAL - 1));

        const inner = document.getElementById('pmSliderInner');
        const prev  = document.getElementById('pmPrev');
        const next  = document.getElementById('pmNext');
        if (!inner) return;

        const slideW = inner.querySelector('.pm-slide-item')?.offsetWidth || 0;
        inner.style.transform = 'translateX(-' + (idx * (slideW + 8)) + 'px)';

        if (prev) prev.disabled = idx === 0;

      };
```

이 부분은 삭제해도 슬라이더가 정상 작동해요! 왜냐하면 같은 기능이 345~361번째 줄의 `setTimeout` 안에서 이미 구현되어 있거든요.

## 3-10. Tailwind CSS 활용 개선 (중요도: 낮음)

### 문제

`myinfoUI.js`, `mypocketmonUI.js` 등에서 inline style을 많이 사용하고 있어요.

### 해결 방법

이건 한 번에 다 바꿀 필요 없어요! **새 코드를 작성할 때부터** Tailwind를 쓰는 습관을 들이면 돼요. 기존 코드는 시간이 있을 때 조금씩 바꿔보세요.

**자주 쓰는 style ↔ Tailwind 변환표:**

| inline style | Tailwind 클래스 |
|---|---|
| `display: flex` | `flex` |
| `flex-direction: column` | `flex-col` |
| `align-items: center` | `items-center` |
| `justify-content: center` | `justify-center` |
| `gap: 16px` | `gap-4` |
| `padding: 24px` | `p-6` |
| `padding: 24px 16px` | `px-4 py-6` |
| `border-radius: 14px` | `rounded-xl` |
| `background: #FFF` | `bg-white` |
| `width: 100%` | `w-full` |
| `font-size: 14px` | `text-sm` |
| `font-size: 18px` | `text-lg` |
| `font-weight: 600` | `font-semibold` |
| `color: #1a3a35` | `text-[#1a3a35]` |
| `margin-bottom: 24px` | `mb-6` |

**변환 예시:**

```html
<!-- 수정 전 -->
<div style="display:flex; width:100%; padding:24px 16px; flex-direction:column; 
     align-items:center; gap:16px; border-radius:14px; background:#FFF; 
     box-shadow:0 1px 3px 0 rgba(0,0,0,0.10), 0 1px 2px -1px rgba(0,0,0,0.10);">

<!-- 수정 후 -->
<div class="flex w-full px-4 py-6 flex-col items-center gap-4 rounded-xl bg-white shadow">
```

훨씬 짧고 읽기 쉽죠?

> **팁:** Tailwind CSS 공식 문서(https://tailwindcss.com/docs)에서 검색하면 어떤 CSS 속성이 어떤 클래스에 대응하는지 쉽게 찾을 수 있어요!

```js
// 수정 전 (src/components/board/board.js)
const getPosts = async () => {
  try {
    const response = await fetch(`${BASE_URL}/posts`, { method: 'GET' });
    if (!response.ok) throw new Error('Failed to fetch posts');
    const data = await response.json();
    const posts = data.data?.content ?? data.data ?? dummyData;
    return Array.isArray(posts) ? posts : dummyData;
  } catch (error) {
    console.error('Error fetching posts:', error);
    return dummyData;
  }
};
```
