# 프로젝트 코드 리뷰

> 리뷰 날짜: 2026-03-30
> 대상: 포켓아카이브 (Pocket Archive) — 포켓몬 도감과 커뮤니티 기능을 함께 제공하는 Vanilla JS 웹 서비스

---

## 1. 프로젝트 구조 살펴보기

현재 프로젝트의 파일 구조는 이렇게 되어 있습니다:

```
pocket-archive/
├── .github/
│   └── pull_request_template.md   ← PR 템플릿
├── public/
│   ├── favicon.svg                ← 파비콘
│   └── icons.svg                  ← 아이콘 스프라이트
├── src/
│   └── assets/
│       ├── hero.png               ← 히어로 이미지
│       ├── javascript.svg         ← JS 로고
│       └── vite.svg               ← Vite 로고
├── .gitignore
├── api.md                         ← API 문서
├── index.html                     ← 진입점 HTML
├── package.json
├── package-lock.json
├── README.md
└── vite.config.js                 ← Vite + Tailwind CSS 설정
```

아직 프로젝트 초기 단계라서 실제 JavaScript 소스 파일(`src/main.js`)이나 CSS 파일은 없는 상태예요. 지금은 프로젝트 뼈대(보일러플레이트)만 세팅된 상태입니다.

## 2. 잘한 점

### Vite + Tailwind CSS v4 조합

최신 빌드 도구인 Vite와 Tailwind CSS v4를 `@tailwindcss/vite` 플러그인으로 연결한 것은 아주 좋은 선택이에요! CDN 방식 대신 빌드 도구 플러그인을 사용하면 개발 중에 더 빠르고 정확한 스타일링이 가능해요.

```js
// vite.config.js
import tailwindcss from '@tailwindcss/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [tailwindcss()],
})
```

이 설정이 깔끔하고 정확해요. 잘했습니다!

### PR 템플릿 구성

`.github/pull_request_template.md` 파일을 만들어 놓은 것은 팀 협업에서 정말 중요한 부분이에요. 체크리스트, 테스트 방법, 커밋 규칙까지 잘 정리되어 있어요. 팀원들이 일관된 방식으로 PR을 작성할 수 있어서 코드 리뷰가 훨씬 수월해질 거예요.

### API 문서 사전 작성

`api.md`에 회원가입, 로그인, 게시글 CRUD 등 API 구조를 미리 문서화한 것은 정말 훌륭해요! 개발을 시작하기 전에 API 설계를 먼저 정리하면 프론트엔드와 백엔드 간의 소통이 훨씬 원활해져요. HTTP 상태 코드 표도 잘 정리되어 있어요.

### .gitignore 설정

`.gitignore` 파일이 잘 설정되어 있어서 `node_modules/`, `dist/`, `.env` 같은 파일들이 Git에 올라가지 않도록 잘 관리되고 있어요. 이건 팀 프로젝트에서 아주 기본적이면서도 중요한 부분인데, 꼼꼼하게 해놓았네요!

### README에 팀 소개 및 실행 방법 포함

README에 팀원 소개, 프로젝트 설명, 설치 및 실행 방법이 포함되어 있어요. 다른 사람이 이 프로젝트를 처음 보더라도 어떻게 시작해야 하는지 알 수 있어요. 좋은 습관이에요!

## 3. 개선하면 좋을 점

### 3-1. 프로젝트 이름 오타 수정 (중요도: 높음)

**현재 상태:**

`package.json`과 `index.html`의 `<title>` 태그에 프로젝트 이름이 **"pocket-archove"** 로 되어 있어요. "archive"가 아니라 "archove"로 오타가 나 있어요.

```json
// package.json 2번째 줄
"name": "pocket-archove"
```

```html
<!-- index.html 6번째 줄 -->
<title>pocket-archove</title>
```

**왜 개선하면 좋을까?**

`package.json`의 `name`은 프로젝트를 식별하는 중요한 값이에요. 오타가 있으면 나중에 배포하거나 다른 도구와 연동할 때 혼란이 생길 수 있어요. `<title>` 태그는 브라우저 탭에 표시되는 텍스트라서 사용자에게 직접 보이는 부분이에요.

**추천:**

```json
// package.json
"name": "pocket-archive"
```

```html
<!-- index.html -->
<title>포켓아카이브</title>
```

`<title>`에는 한글로 서비스 이름을 넣어주면 사용자에게 더 친근하게 다가갈 수 있어요.

---

### 3-2. `src/main.js` 파일 누락 (중요도: 높음)

**현재 상태:**

`index.html`에서 `src/main.js` 파일을 불러오고 있지만, 실제로 이 파일이 존재하지 않아요.

```html
<!-- index.html 10번째 줄 -->
<script type="module" src="/src/main.js"></script>
```

하지만 `src/` 폴더에는 `assets/` 폴더만 있고 `main.js` 파일이 없어요.

**왜 개선하면 좋을까?**

이 상태에서 `npm run dev`로 프로젝트를 실행하면, 브라우저 콘솔에 `main.js`를 찾을 수 없다는 에러가 발생해요. 프로젝트가 정상적으로 동작하려면 이 진입점(entry point) 파일이 반드시 있어야 해요.

**추천:**

`src/main.js` 파일을 만들어주세요. Tailwind CSS v4를 사용하려면 CSS import도 함께 해야 해요:

```js
// src/main.js
import './style.css'

console.log('포켓아카이브 시작!')
```

```css
/* src/style.css */
@import 'tailwindcss';
```

이렇게 하면 Tailwind CSS가 프로젝트에 적용되고, 개발을 시작할 준비가 완료돼요!

---

### 3-3. `<html lang="en">`을 `lang="ko"`로 변경 (중요도: 중간)

**현재 상태:**

```html
<!-- index.html 2번째 줄 -->
<html lang="en">
```

**왜 개선하면 좋을까?**

`lang` 속성은 이 웹페이지가 어떤 언어로 작성되어 있는지 브라우저와 스크린 리더(시각장애인용 화면 읽기 도구)에게 알려주는 역할을 해요. 포켓아카이브는 한국어 서비스니까 `"ko"`로 설정하는 것이 맞아요.

이 값이 잘못되어 있으면:
- 스크린 리더가 한국어 텍스트를 영어 발음으로 읽으려고 시도할 수 있어요
- 브라우저의 자동 번역 기능이 잘못 작동할 수 있어요
- 웹 접근성(accessibility) 검사에서 경고가 나올 수 있어요

**추천:**

```html
<html lang="ko">
```

### 3-4. Vite 멀티 페이지 설정 고려 (중요도: 중간)

**현재 상태:**

현재 `vite.config.js`에는 멀티 페이지 설정이 없어요.

```js
// vite.config.js
export default defineConfig({
  plugins: [tailwindcss()],
})
```

**왜 개선하면 좋을까?**

`api.md`를 보면 로그인, 회원가입, 게시판, 포켓몬 도감 등 여러 페이지가 필요한 프로젝트예요. Vanilla JS로 멀티 페이지 앱(MPA)을 만들 때는 Vite에 각 페이지의 HTML 파일을 등록해줘야 빌드할 때 모든 페이지가 포함돼요.

**추천:**

나중에 페이지를 추가할 때 `vite.config.js`에 아래처럼 설정해주세요:

```js
import { resolve } from 'node:path'
import tailwindcss from '@tailwindcss/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        login: resolve(__dirname, 'src/pages/login/index.html'),
        // 페이지가 추가될 때마다 여기에 등록
      },
    },
  },
})
```

지금 당장은 아니더라도, 페이지를 만들기 시작할 때 꼭 기억해두세요!

---

### 3-5. `<title>` 태그 내용 개선 (중요도: 낮음)

**현재 상태:**

```html
<title>pocket-archove</title>
```

**왜 개선하면 좋을까?**

`<title>` 태그는 브라우저 탭, 검색엔진 결과, 북마크 이름에 사용돼요. 서비스의 정체성을 드러내는 중요한 요소이므로, 프로젝트 이름을 정확하고 의미 있게 적어주면 좋아요.

**추천:**

```html
<title>포켓아카이브 - 나만의 포켓몬 도감</title>
```

각 페이지마다 다른 제목을 넣어주면 더 좋아요. 예를 들어 로그인 페이지라면:
```html
<title>로그인 - 포켓아카이브</title>
```
