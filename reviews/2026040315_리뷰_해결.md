## 3-1. `<postlist>` 커스텀 태그 대신 표준 HTML 태그 사용 (중요도: 높음)

### 문제

`pages/board.html` 148번째 줄에 HTML 표준에 없는 `<postlist>` 태그를 사용하고 있고, `src/components/board.js` 118번째 줄에서 이를 참조하고 있어요.

### 해결 방법

**Step 1.** `pages/board.html` 수정하기

```html
<!-- 수정 전 (pages/board.html 148번째 줄) -->
<postlist></postlist>

<!-- 수정 후 -->
<div id="postlist"></div>
```

**Step 2.** `src/components/board.js` 수정하기

```js
// 수정 전 (src/components/board.js 118번째 줄)
const postlist = document.querySelector('postlist');

// 수정 후
const postlist = document.getElementById('postlist');
```

> **쉽게 말하면:** HTML에 없는 태그를 만들어 쓰는 것은 존재하지 않는 교통 표지판을 세워놓는 것과 같아요. 운전자(브라우저)가 무슨 뜻인지 몰라요. 표준 `<div>` 태그에 이름표(`id`)를 달아주는 게 누구나 이해할 수 있는 방법이에요.

## 3-2. `menuBtn` 이벤트 리스너 중복 등록 (중요도: 높음)

### 문제

`src/main.js`에서 `menuBtn`의 클릭 이벤트를 16~25번째 줄과 87~112번째 줄(`initSidebar` 함수)에서 **두 번** 등록하고 있어요.

### 해결 방법

**Step 1.** `src/main.js` 16~25번째 줄 삭제하기

아래 코드를 모두 지워주세요:

```js
// 수정 전 (src/main.js 16~25번째 줄) — 이 부분을 삭제
const menuBtn = document.getElementById('menuBtn');
const sidebar = document.getElementById('sidebar');
const overlay = document.getElementById('overlay');

if (menuBtn) {
  menuBtn.addEventListener('click', () => {
    sidebar.classList.add('active');
```

수정 후 `src/main.js`의 앞부분은 이렇게 되어야 해요:

```js
// 수정 후 (src/main.js)
import { Header } from './components/header.js';

console.log('포켓아카이브 실행중');

    <div id="content" class="content"></div>

  ${Footer()}
`;

// 16~25번째 줄의 menuBtn 코드 삭제됨
// initSidebar()에서 통합 관리

async function loadPage() {
  // ... (기존 코드 유지)
}
```

> **쉽게 말하면:** 같은 초인종 버튼에 벨을 2개 달아놓은 거예요. 누르면 "딩동! 딩동!" 두 번 울려요. 하나만 남기면 깔끔하게 "딩동!" 한 번만 울려요.

## 3-3. `innerHTML`에 사용자 데이터 삽입 시 XSS 위험 (중요도: 높음)

### 문제

`src/components/board.js` 138번째 줄에서 게시글 데이터를 `innerHTML`에 직접 삽입하고 있어요. 나중에 API에서 실제 데이터를 받아올 때 XSS 공격에 취약해질 수 있어요.

### 해결 방법

지금 당장은 더미 데이터를 사용하고 있어서 급하지 않지만, **API를 연동할 때 반드시** 아래 방식을 적용해주세요.

**Step 1.** 안전하게 텍스트를 삽입하는 헬퍼 함수 만들기

`src/components/board.js` 맨 위에 추가해주세요:

```js
// 수정 후 (src/components/board.js 맨 위에 추가)
function escapeHTML(str) {
  const div = document.createElement('div');
  div.textContent = str;
  return div.innerHTML;
}
```

**Step 2.** `innerHTML`에 사용자 데이터를 넣을 때 `escapeHTML()` 사용

```js
// 수정 전 (src/components/board.js 138번째 줄 부근)
postElement.innerHTML = `
  <span ...>${post.category}</span>
  <p ...>${post.title}</p>
  <span ...>${post.user}</span>
`;

// 수정 후
postElement.innerHTML = `
  <span ...>${escapeHTML(post.category)}</span>
  <p ...>${escapeHTML(post.title)}</p>
  <span ...>${escapeHTML(post.user)}</span>
`;
```

이렇게 하면 `<script>` 같은 위험한 태그가 그냥 텍스트로 표시돼요.

> **쉽게 말하면:** 편지를 받았을 때 내용을 그대로 읽는 것(`innerHTML`)은 위험할 수 있어요. 혹시 "이 종이를 태워라"라고 적혀 있으면 곤란하잖아요. `escapeHTML()`은 편지를 유리창 너머로 보여주는 것과 같아요 — 내용은 보이지만 위험한 행동은 할 수 없어요.

## 3-5. `board.html` HTML 조각 안에 `<link>` 태그 제거 (중요도: 중간)

### 문제

`pages/board.html` 1번째 줄에 `<link rel="stylesheet" ...>` 태그가 있어요. 이 파일은 HTML 조각으로 사용되므로 `<link>` 태그가 있으면 안 돼요.

### 해결 방법

**Step 1.** `pages/board.html` 1~2번째 줄 삭제

```html
<!-- 수정 전 (pages/board.html 1~2번째 줄) — 이 줄들을 삭제 -->
<link rel="stylesheet" href="/src/style.css" />
<!-- Start body element -->
```

Tailwind CSS는 이미 `index.html` → `main.js` → `style.css` 경로로 불러오고 있어서, 여기서 다시 불러올 필요가 없어요.

> **쉽게 말하면:** 퍼즐 한 조각에 액자 프레임까지 붙여놓은 거예요. 조각만 있으면 돼요 — 프레임은 전체 그림(`index.html`)에 이미 있으니까요.

## 3-6. 인라인 `style` 속성을 Tailwind 클래스로 전환 (중요도: 중간)

### 문제

`pages/board.html`에서 같은 요소에 인라인 `style`과 Tailwind 클래스를 동시에 사용하고 있어서 스타일 관리가 어려워요.

### 해결 방법

주요 인라인 스타일을 Tailwind 클래스로 변환하는 방법을 알려드릴게요.

**Step 1.** 카테고리 영역 (14~16번째 줄)

```html
<!-- 수정 전 -->
<div
  class="shrink-0 bg-white rounded-xl ..."
  style="display: flex; flex-direction: column; align-items: flex-start; width: 320px; height: 370px; padding: 24px 24px 0 24px; gap: 16px"
>

<!-- 수정 후 — style을 Tailwind로 변환 -->
<div
  class="shrink-0 bg-white rounded-xl flex flex-col items-start w-80 h-[370px] pt-6 px-6 gap-4 ..."
>
```

**자주 쓰는 변환표:**

| 인라인 style | Tailwind 클래스 |
|---|---|
| `display: flex` | `flex` |
| `flex-direction: column` | `flex-col` |
| `align-items: flex-start` | `items-start` |
| `align-items: center` | `items-center` |
| `justify-content: center` | `justify-center` |
| `gap: 16px` | `gap-4` |
| `padding: 24px` | `p-6` |
| `width: 100%` | `w-full` |
| `min-height: 78px` | `min-h-[78px]` |

> **쉽게 말하면:** 옷에 직접 "빨간색"이라고 낙서하는 것(인라인 style)보다, "빨간색" 라벨(Tailwind 클래스)을 붙이는 게 나중에 색을 바꿀 때 훨씬 편해요.

## 3-7. 동적 `import()` 순서 수정 (중요도: 중간)

### 문제

`src/main.js` 49~53번째 줄에서 HTML 삽입 전에 `board.js`를 import하고 있어서, board.js가 DOM 요소를 찾지 못할 위험이 있어요.

### 해결 방법

**Step 1.** `src/main.js` 49~53번째 줄 순서 변경

```js
// 수정 전 (src/main.js 49~53번째 줄)
if (current === 'board') {
  import('./components/board.js');
}

document.getElementById('content').innerHTML = html;

// 수정 후 — HTML 먼저 넣고, 그 다음 import
document.getElementById('content').innerHTML = html;

if (current === 'board') {
  await import('./components/board.js');
}
```

`await`를 붙이면 `board.js`가 완전히 실행된 후에 다음 코드로 넘어가요. 이렇게 하면 순서가 확실하게 보장돼요:
1. HTML이 DOM에 들어감
2. board.js가 DOM 요소에 접근

> **쉽게 말하면:** 가구(HTML)를 방에 넣기 전에 인테리어(board.js)를 시작하는 것과 같아요. 가구를 먼저 넣고 인테리어를 하는 게 순서에 맞아요.

## 3-9. 검색 필터 `filter-type` 기능 연결 (중요도: 낮음)

### 문제

`pages/board.html`에 검색 유형 `<select>`가 있지만 `src/components/board.js`에서 이 값을 사용하지 않아요.

### 해결 방법

**Step 1.** `src/components/board.js` 238~246번째 줄 수정

```js
// 수정 전 (src/components/board.js 238~244번째 줄)
document.getElementById('search-btn')?.addEventListener('click', () => {
  const keyword = document.getElementById('search-input').value.trim().toLowerCase();
  if (!keyword) {
    renderPosts(dummyData);
    return;
  }
  const filtered = dummyData.filter((post) =>
    post.title.toLowerCase().includes(keyword) ||
    post.user.toLowerCase().includes(keyword)
  );
  renderPosts(filtered);
});

// 수정 후
document.getElementById('search-btn')?.addEventListener('click', () => {
  const keyword = document.getElementById('search-input').value.trim().toLowerCase();
  if (!keyword) {
    renderPosts(dummyData);
    return;
  }

  const filterType = document.getElementById('filter-type').value;
  const filtered = dummyData.filter((post) => {
    if (filterType === 'title') {
      return post.title.toLowerCase().includes(keyword);
    }
    if (filterType === 'author') {
      return post.user.toLowerCase().includes(keyword);
    }
    return false;
  });
  renderPosts(filtered);
});
```

> **쉽게 말하면:** 검색 필터 버튼은 만들어놨는데 연결 안 된 스위치 같아요. 전선을 연결해서 제대로 동작하게 만들어주는 거예요.

## 3-10. 사이드바 `.active` 스타일 CSS 중복 제거 (중요도: 낮음)

### 문제

`src/styles/layout.css`에서 사이드바 `.active` 스타일이 182번째 줄과 202번째 줄에 중복으로 작성되어 있어요.

### 해결 방법

**Step 1.** `src/styles/layout.css` 182~192번째 줄에서 중첩된 `.active` 부분만 삭제

```css

.sidebar-nav a {
  padding: 16px 20px;
  display: block;
  text-decoration: none;
  color: #333;
  .active {            /* ← 이 중첩 부분을 삭제 */
    background: #cde5ec;
    border-radius: 12px;
    font-weight: bold;
  }

  padding: 16px 20px;
  display: block;
  text-decoration: none;

  border-radius: 12px;
```

> **쉽게 말하면:** 같은 안내판을 건물 입구에 2개 붙여놓은 거예요. 하나만 있으면 충분해요!
