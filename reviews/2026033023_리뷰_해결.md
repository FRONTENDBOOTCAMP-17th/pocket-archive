## 3-1. 프로젝트 이름 오타 수정 (중요도: 높음)

### 문제

`package.json` 2번째 줄과 `index.html` 6번째 줄에 프로젝트 이름이 "pocket-archove"로 잘못 적혀 있어요. 올바른 이름은 "pocket-archive"예요.

### 해결 방법

**Step 1.** `package.json` 수정하기

```json
// 수정 전 (package.json 2번째 줄)
"name": "pocket-archove",

// 수정 후
"name": "pocket-archive",
```

**Step 2.** `index.html` 수정하기

```html
<!-- 수정 전 (index.html 6번째 줄) -->
<title>pocket-archove</title>

<!-- 수정 후 -->
<title>포켓아카이브</title>
```

> **쉽게 말하면:** 내 이름이 김철수인데 명찰에 "김천수"라고 적혀 있는 것과 같아요. 이름을 정확하게 고쳐주면 돼요!

## 3-2. `src/main.js` 파일 생성 (중요도: 높음)

### 문제

`index.html` 10번째 줄에서 `src/main.js`를 불러오고 있지만 이 파일이 존재하지 않아요. 프로젝트를 실행하면 브라우저 콘솔에 에러가 발생해요.

### 해결 방법

**Step 1.** `src/main.js` 파일 새로 만들기

아래 내용으로 `src/main.js` 파일을 생성해주세요:

```js
// src/main.js (새로 만드는 파일)
import './style.css'

const app = document.querySelector('#app')
app.textContent = '포켓아카이브에 오신 것을 환영합니다!'
```

**Step 2.** `src/style.css` 파일 새로 만들기

Tailwind CSS가 동작하려면 CSS 파일에서 Tailwind를 import 해야 해요. 아래 내용으로 `src/style.css` 파일을 생성해주세요:

```css
/* src/style.css (새로 만드는 파일) */
@import 'tailwindcss';
```

**Step 3.** 확인하기

터미널에서 아래 명령어를 실행해보세요:

```bash
npm run dev
```

브라우저에 "포켓아카이브에 오신 것을 환영합니다!"가 나타나면 성공이에요!

> **쉽게 말하면:** `index.html`이 "main.js야, 일 좀 해줘!"라고 부르고 있는데, 정작 main.js가 없어서 아무 일도 안 일어나는 거예요. 파일을 만들어서 "네, 여기 있어요!" 하고 응답해주면 돼요.

## 3-3. `<html lang="en">`을 `lang="ko"`로 변경 (중요도: 중간)

### 문제

`index.html` 2번째 줄에서 `lang="en"`으로 되어 있어요. 한국어 서비스인데 영어로 설정되어 있는 거예요.

### 해결 방법

**Step 1.** `index.html` 수정하기

```html
<!-- 수정 전 (index.html 2번째 줄) -->
<html lang="en">

<!-- 수정 후 -->
<html lang="ko">
```

이것만 바꿔주면 끝이에요! 아주 간단하죠?

> **쉽게 말하면:** 한국 식당인데 간판에 "English Restaurant"이라고 적혀 있는 것과 비슷해요. "한국 식당"이라고 바꿔주면 손님(브라우저, 스크린 리더)이 헷갈리지 않아요.

## 3-4. Vite 멀티 페이지 설정 고려 (중요도: 중간)

### 문제

프로젝트가 여러 페이지(로그인, 회원가입, 게시판, 도감 등)를 가질 예정인데, 현재 Vite 설정에는 멀티 페이지 빌드 설정이 없어요.

### 해결 방법

이 항목은 **지금 당장 하지 않아도 돼요!** 페이지를 추가하기 시작할 때 적용하면 됩니다.

**Step 1.** 페이지를 만들기 시작할 때 폴더 구조 잡기

```
src/
├── pages/
│   ├── login/
│   │   └── index.html
│   ├── register/
│   │   └── index.html
│   └── pokedex/
│       └── index.html
├── style.css
└── main.js
```

**Step 2.** `vite.config.js` 수정하기

```js
// 수정 전 (vite.config.js)
import tailwindcss from '@tailwindcss/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [tailwindcss()],
})

// 수정 후
import { resolve } from 'node:path'
import tailwindcss from '@tailwindcss/vite'
import { defineConfig } from 'vite'

export default defineConfig({
  plugins: [tailwindcss()],
  build: {
    rollupOptions: {
      input: {
        main: resolve(__dirname, 'index.html'),
        login: resolve(__dirname, 'src/pages/login/index.html'),
        register: resolve(__dirname, 'src/pages/register/index.html'),
        pokedex: resolve(__dirname, 'src/pages/pokedex/index.html'),
      },
    },
  },
})
```

> **쉽게 말하면:** 책을 만들 때 "목차"를 적어두는 거예요. Vite에게 "이 프로젝트에는 이런 페이지들이 있어"라고 알려줘야 빌드할 때 모든 페이지를 포함시켜줘요.

## 3-5. `<title>` 태그 내용 개선 (중요도: 낮음)

### 문제

`index.html` 6번째 줄의 `<title>` 태그에 오타가 있고, 서비스 이름으로서 의미가 부족해요.

### 해결 방법

**Step 1.** `index.html` 수정하기 (3-1 해결과 함께)

```html
<!-- 수정 전 (index.html 6번째 줄) -->
<title>pocket-archove</title>

<!-- 수정 후 -->
<title>포켓아카이브 - 나만의 포켓몬 도감</title>
```

이후에 다른 페이지를 만들 때도 각 페이지에 맞는 제목을 넣어주세요:

```html
<!-- 로그인 페이지 -->
<title>로그인 - 포켓아카이브</title>

<!-- 회원가입 페이지 -->
<title>회원가입 - 포켓아카이브</title>

<!-- 도감 페이지 -->
<title>포켓몬 도감 - 포켓아카이브</title>
```

> **쉽게 말하면:** 브라우저 탭에 보이는 이름이에요. "Document"보다 "네이버"가 어떤 사이트인지 바로 알 수 있는 것처럼, 의미 있는 제목을 넣어주면 좋아요.
