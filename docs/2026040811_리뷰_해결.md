## 3-1. register.js — 중복확인 없이 회원가입이 가능한 버그 (중요도: 높음)

### 문제

`src/components/Auth/register.js`의 `initRegister()` 함수 안에서 `validNickname`과 `validId`의 **초기값이 `true`**로 되어 있어요. 입력값을 바꿀 때도 `true`로 리셋해요. 그래서 중복확인을 한 번도 하지 않아도 회원가입 버튼이 활성화돼요.

### 해결 방법

**Step 1.** 초기값을 `false`로 변경하세요.

```js
// 수정 전 (register.js 136~137번째 줄)
  let validNickname = true;
  let validId = true;

// 수정 후
  let validNickname = false;
  let validId = false;
```

**Step 2.** 입력값 변경 시 `false`로 리셋하세요.

```js
// 수정 전 (register.js 199~207번째 줄)
  nickname.addEventListener('input', () => {
    validNickname = true;
    updateButtonState();
  });

  id.addEventListener('input', () => {
    validId = true;
    updateButtonState();
  });

// 수정 후
  nickname.addEventListener('input', () => {
    validNickname = false;
    updateButtonState();
  });

  id.addEventListener('input', () => {
    validId = false;
    updateButtonState();
  });
```

**Step 3.** `updateButtonState`에서 `=== true` 대신 간결하게 작성하세요 (선택).

```js
// 수정 전 (register.js 233~240번째 줄)
  function updateButtonState() {
    const signupBtn = document.getElementById('signupBtn');
    if (validNickname === true && validId === true) {
      signupBtn.disabled = false;
    } else {
      signupBtn.disabled = true;
    }
  }

// 수정 후
  function updateButtonState() {
    const signupBtn = document.getElementById('signupBtn');
    signupBtn.disabled = !(validNickname && validId);
  }
```

**왜 이렇게 고쳐야 하나요?**

`validNickname`과 `validId`는 "중복확인을 완료했고 사용 가능하다"를 뜻하는 플래그(flag)예요.
- 처음에는 아직 확인 안 했으니 `false`
- 중복확인 API 호출 후 사용 가능하면 `true`
- 입력값을 바꾸면 이전 확인 결과가 무효가 되니 다시 `false`

> **쉽게 말하면:** "통과" 도장은 검사를 받은 후에만 찍혀야 해요. 처음부터 도장이 찍혀있거나, 다시 쓸 때 자동으로 도장이 찍히면 검사가 의미 없어져요.

## 3-2. boardDetailUI.js — comment.content와 post.title에 escapeHtml 적용 (중요도: 높음)

### 문제

`src/components/board/boardDetailUI.js`에서 댓글 내용(`comment.content`, 37번째 줄)과 게시글 제목(`post.title`, 69번째 줄)에 `escapeHtml`이 적용되지 않아서 XSS 공격에 취약해요.

### 해결 방법

**Step 1.** 댓글 내용에 `escapeHtml` 적용하세요.

```js
// 수정 전 (boardDetailUI.js 37번째 줄)
      <p id="comment-content-${comment.commentId}" class="text-gray-600 text-[14px] text-left leading-relaxed whitespace-pre-wrap">${comment.content}</p>

// 수정 후
      <p id="comment-content-${comment.commentId}" class="text-gray-600 text-[14px] text-left leading-relaxed whitespace-pre-wrap">${escapeHtml(comment.content)}</p>
```

**Step 2.** 게시글 제목에 `escapeHtml` 적용하세요.

```js
// 수정 전 (boardDetailUI.js 69번째 줄)
          <h1 class="text-3xl font-bold text-[#1a3a35] leading-tight">
            ${post.title}
          </h1>

// 수정 후
          <h1 class="text-3xl font-bold text-[#1a3a35] leading-tight">
            ${escapeHtml(post.title)}
          </h1>
```

**왜 이렇게 고쳐야 하나요?**

`comment.content`는 **다른 사용자가 작성한 텍스트**예요. 이걸 `innerHTML`에 그대로 넣으면, 누군가 댓글에 `<script>` 태그를 넣어서 다른 사용자의 브라우저에서 코드를 실행시킬 수 있어요. 이것이 **XSS(Cross-Site Scripting)** 공격이에요.

`escapeHtml()`은 `<`를 `&lt;`로, `>`를 `&gt;`로 바꿔서 HTML 태그가 실행되지 않게 만들어요. 화면에는 문자 그대로 보이고, 코드로 실행되지는 않아요.

> **쉽게 말하면:** 편지를 읽을 때, 누군가 편지 안에 "이 편지를 받는 사람의 지갑을 열어라"라고 적어놨다면 무시해야 해요. `escapeHtml`은 그런 위험한 지시를 무시하게 해주는 안전장치예요.

## 3-3. boardDetail.js — 댓글 수정 시 onclick에 originalContent 직접 삽입 (중요도: 높음)

### 문제

`src/components/board/boardDetail.js` 122~140번째 줄에서 `originalContent`를 `onclick` 속성에 직접 넣고 있어요. 댓글 내용에 `'`(작은따옴표)가 포함되면 HTML이 깨지고, 악의적인 내용이면 XSS가 가능해요.

### 해결 방법

**Step 1.** `escapeHtml`을 import 하세요 (이미 되어있다면 건너뛰세요).

```js
// boardDetail.js 맨 위에 추가 (없는 경우)
import { escapeHtml } from '../../utils/escapeHtml.js';
```

**Step 2.** `toggleEditMode` 함수를 수정하세요.

```js
// 수정 전 (boardDetail.js 122~141번째 줄)
window.toggleEditMode = (commentId) => {
  const contentP = document.getElementById(`comment-content-${commentId}`);
  const btnGroup = document.getElementById(`comment-btns-${commentId}`);
  const originalContent = contentP.innerText;

  // 1. 내용 영역을 textarea로 교체
  contentP.innerHTML = `
    <textarea id="edit-input-${commentId}" 
      class="w-full p-3 mt-2 border border-gray-200 rounded-lg focus:outline-none focus:border-[#05B29F] text-[14px] resize-none"
      rows="3">${originalContent}</textarea>
  `;

  // 2. 버튼을 '저장/취소'로 교체
  btnGroup.innerHTML = `
    <button onclick="saveEditComment(${commentId}, '${originalContent}')" 
            class="text-[11px] text-[#05B29F] font-bold">저장</button>
    <button onclick="cancelEditMode(${commentId}, '${originalContent}')" 
            class="text-[11px] text-gray-400 font-bold">취소</button>
  `;
};

// 수정 후
window.toggleEditMode = (commentId) => {
  const contentP = document.getElementById(`comment-content-${commentId}`);
  const btnGroup = document.getElementById(`comment-btns-${commentId}`);
  const originalContent = contentP.innerText;

  // 1. 내용 영역을 textarea로 교체 (escapeHtml로 안전하게!)
  contentP.innerHTML = `
    <textarea id="edit-input-${commentId}" 
      class="w-full p-3 mt-2 border border-gray-200 rounded-lg focus:outline-none focus:border-[#05B29F] text-[14px] resize-none"
      rows="3">${escapeHtml(originalContent)}</textarea>
  `;

  // 2. 버튼을 '저장/취소'로 교체 (onclick 대신 addEventListener!)
  btnGroup.innerHTML = `
    <button id="save-btn-${commentId}" class="text-[11px] text-[#05B29F] font-bold">저장</button>
    <button id="cancel-btn-${commentId}" class="text-[11px] text-gray-400 font-bold">취소</button>
  `;

  document.getElementById(`save-btn-${commentId}`).addEventListener('click', () => {
    saveEditComment(commentId, originalContent);
  });
  document.getElementById(`cancel-btn-${commentId}`).addEventListener('click', () => {
    location.reload();
  });
};
```

**핵심 변경사항:**
1. `textarea` 안의 `originalContent` → `escapeHtml(originalContent)` 감싸기
2. `onclick="saveEditComment(${commentId}, '${originalContent}')"` → `addEventListener`로 변경
3. `onclick="cancelEditMode(${commentId}, '${originalContent}')"` → `addEventListener`로 변경

**왜 이렇게 고쳐야 하나요?**

`onclick` 속성에 변수를 넣으면 그 변수가 **HTML 문자열의 일부**가 돼요. 작은따옴표, 큰따옴표, `<`, `>` 같은 특수문자가 들어가면 HTML 구조가 깨져요.

`addEventListener`를 사용하면 `originalContent`는 **JavaScript 변수 그대로** 전달되기 때문에, 어떤 특수문자가 있어도 안전해요.

> **쉽게 말하면:** 이름표에 이름을 직접 적으면(`onclick`) 이름에 특수문자가 있을 때 이름표가 찢어질 수 있어요. 대신 이름표에 번호만 적고, 번호로 이름을 찾아보면(`addEventListener`) 어떤 이름이든 안전해요.

## 3-4. boardDetail.js + pokedex.js — window 전역 함수 → 이벤트 위임 패턴 (중요도: 높음)

### 문제

`boardDetail.js`에 `window.*` 전역 함수 6개, `pokedex.js`에 3개, `pokedexUI.js`에 1개 — 총 10개의 전역 함수가 있어요. HTML의 `onclick` 속성과 결합되어 결합도가 높아요.

### 해결 방법

이건 큰 리팩토링이라 **boardDetail.js부터 하나씩** 바꿔보세요.

**Step 1.** `boardDetailUI.js`의 Comment 함수에서 `onclick` → `data-action` 변경

```js
// 수정 전 (boardDetailUI.js 27~30번째 줄)
              <button onclick="toggleEditMode(${comment.commentId})" 
                      class="text-[11px] text-gray-400 hover:text-[#05B29F] font-bold transition-colors">수정</button>
              <button onclick="handleDeleteComment(${comment.commentId})" 
                      class="text-[11px] text-gray-400 hover:text-red-500 font-bold transition-colors">삭제</button>

              <button data-action="edit-comment" data-comment-id="${comment.commentId}" 
                      class="text-[11px] text-gray-400 hover:text-[#05B29F] font-bold transition-colors cursor-pointer">수정</button>
              <button data-action="delete-comment" data-comment-id="${comment.commentId}" 
                      class="text-[11px] text-gray-400 hover:text-red-500 font-bold transition-colors cursor-pointer">삭제</button>
```

**Step 2.** `BoardDetailContent`에서도 `onclick` → `data-action` 변경

```js
// 수정 전 (boardDetailUI.js 57~58번째 줄)
        ? `<div class="flex gap-3"><button onclick="handleEditPost(${post.postId})" class="text-sm text-gray-400 hover:text-[#05B29F] font-bold transition-colors">수정</button>
                <button onclick="handleDeletePost(${post.postId})" class="text-sm text-gray-400 hover:text-red-500 font-bold transition-colors">삭제</button>

// 수정 후
        ? `<div class="flex gap-3"><button data-action="edit-post" data-post-id="${post.postId}" class="text-sm text-gray-400 hover:text-[#05B29F] font-bold transition-colors cursor-pointer">수정</button>
                <button data-action="delete-post" data-post-id="${post.postId}" class="text-sm text-gray-400 hover:text-red-500 font-bold transition-colors cursor-pointer">삭제</button>
```

**Step 3.** `boardDetail.js`에서 이벤트 위임으로 처리

```js
// 수정 전 — window 전역 함수들
window.toggleEditMode = (commentId) => { ... };
window.cancelEditMode = (commentId, originalContent) => { ... };
window.saveEditComment = async (commentId, oldContent) => { ... };
window.handleDeleteComment = async (commentId) => { ... };
window.handleDeletePost = async (postId) => { ... };
window.handleEditPost = (postId) => { ... };

// 수정 후 — 일반 함수 + 이벤트 위임
function toggleEditMode(commentId) { ... }  // window. 제거!

async function saveEditComment(commentId, oldContent) { ... }  // window. 제거!

async function handleDeleteComment(commentId) { ... }  // window. 제거!

async function handleDeletePost(postId) { ... }  // window. 제거!

function handleEditPost(postId) { ... }  // window. 제거!

// setupCommentEvents에 이벤트 위임 추가 (기존 코드 아래에)
async function setupCommentEvents(postId) {
  // ... 기존 submitBtn 코드 유지 ...

  // 이벤트 위임: 댓글/게시글 액션 버튼 처리
  document.addEventListener('click', async (e) => {
    const btn = e.target.closest('[data-action]');
    if (!btn) return;

    const action = btn.dataset.action;
    const commentId = btn.dataset.commentId;
    const postIdFromBtn = btn.dataset.postId;

    switch (action) {
      case 'edit-comment':
        toggleEditMode(Number(commentId));
        break;
      case 'delete-comment':
        await handleDeleteComment(Number(commentId));
        break;
      case 'edit-post':
        handleEditPost(Number(postIdFromBtn));
        break;
      case 'delete-post':
        await handleDeletePost(Number(postIdFromBtn));
        break;
    }
  });
}
```

**이벤트 위임이 뭔가요?**

보통 이벤트 리스너는 각 버튼마다 하나씩 붙여요:
```js
// 버튼이 10개면 이벤트 리스너도 10개
button1.addEventListener('click', ...);
button2.addEventListener('click', ...);
// ...
```

**이벤트 위임**은 부모 요소 하나에만 리스너를 붙이고, "클릭된 것이 누구인지" 확인하는 패턴이에요:
```js
// 부모에 하나만!
parent.addEventListener('click', (e) => {
  const btn = e.target.closest('[data-action]');  // 클릭된 버튼 찾기
  if (btn.dataset.action === 'delete') { ... }
});
```

장점:
- 동적으로 추가된 요소도 자동으로 처리돼요
- `window` 전역 함수가 필요 없어요
- HTML에 `onclick`을 넣을 필요 없어요

> **쉽게 말하면:** 아파트 우편함에 각 호수마다 우편함이 있는 대신, 경비실에서 모든 우편물을 받아서 호수별로 분류하는 거예요. 새로운 세입자가 와도 경비실 시스템만 있으면 돼요!

## 3-5. myparty.html — `<style>` 태그 CSS → Tailwind 전환 (중요도: 중간)

### 문제

`pages/myparty.html` 1~106번째 줄에 `<style>` 태그로 106줄의 커스텀 CSS가 있어요. 프로젝트가 Tailwind CSS를 사용하므로 원칙에 맞지 않아요.

### 해결 방법

한꺼번에 다 바꾸면 실수가 생길 수 있으니, **핵심 부분부터** 바꿔보세요.

**Step 1.** 메인 그리드 레이아웃을 Tailwind으로 변환

```html
<!-- 수정 전 -->
<style>
  .myparty-trainer-grid {
    display: grid;
    grid-template-columns: 1fr 280px;
    padding: 0 32px 32px 32px;
    gap: 0;
  }
  @media (max-width: 1024px) {
    .myparty-trainer-grid {
      grid-template-columns: 1fr;
      padding: 0 20px 24px 20px;
    }
  }
  @media (max-width: 768px) {
    .myparty-trainer-grid {
      grid-template-columns: 1fr;
      padding: 0 16px 20px 16px;
    }
  }
</style>
<div class="myparty-trainer-grid">

<!-- 수정 후 (Tailwind만 사용) -->
<div class="grid grid-cols-[1fr_280px] px-8 pb-8 max-lg:grid-cols-1 max-lg:px-5 max-lg:pb-6 max-md:px-4 max-md:pb-5">
```

**Step 2.** 포켓몬 카드 그리드를 Tailwind으로 변환

```html
<!-- 수정 전 -->
<style>
  .myparty-pokemon-grid {
    display: grid;
    grid-template-columns: repeat(4, 1fr);
    gap: 16px;
  }
  @media (max-width: 1280px) { grid-template-columns: repeat(3, 1fr); }
  @media (max-width: 768px) { grid-template-columns: repeat(2, 1fr); }
  @media (max-width: 400px) { grid-template-columns: repeat(1, 1fr); }

<div id="pokemon-list" class="myparty-pokemon-grid">

<!-- 수정 후 -->
<div id="pokemon-list" class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4">
```

**Step 3.** 스크롤 컨테이너를 Tailwind으로 변환

```html
<!-- 수정 전 -->
<style>
  .myparty-pokemon-scroll {
    height: 600px;
    overflow-y: auto;
    padding: 0 32px 32px 32px;
  }
</style>
<div class="myparty-pokemon-scroll">

<!-- 수정 후 -->
<div class="h-[600px] overflow-y-auto px-8 pb-8 max-lg:h-[500px] max-lg:px-5 max-md:h-[400px] max-md:px-4">
```

**Step 4.** 스크롤바 스타일만 CSS에 남기세요 (Tailwind으로 커버 안 되는 부분)

Tailwind은 스크롤바 커스터마이징을 기본 지원하지 않아요. 이 부분만 CSS에 남겨도 괜찮아요:

```html
<style>
  /* 스크롤바만 CSS로 (Tailwind으로 안 되는 유일한 부분) */
  .custom-scrollbar { scrollbar-width: thin; scrollbar-color: #d1d5db transparent; }
  .custom-scrollbar::-webkit-scrollbar { width: 6px; }
  .custom-scrollbar::-webkit-scrollbar-thumb { background: #d1d5db; border-radius: 999px; }
</style>
```

**왜 이렇게 고쳐야 하나요?**

Tailwind은 "CSS 파일을 관리하는 부담을 없애자"는 철학이에요. HTML만 보면 스타일이 다 보여요. `<style>` 태그가 섞이면 "이 요소의 스타일이 어디에 있지?" 할 때 HTML 클래스도 보고, `<style>` 태그도 보고, CSS 파일도 봐야 해서 혼란스러워요.

> **쉽게 말하면:** 옷장을 정리할 때 "셔츠는 1번 서랍, 바지는 2번 서랍"으로 규칙을 정했는데, 가끔 셔츠를 3번 서랍에 넣으면 나중에 찾기 어려워요. 스타일은 모두 Tailwind 클래스라는 "1번 서랍"에 넣는 게 좋아요.

## 3-6. api/config.js를 실제로 사용하기 (중요도: 중간)


**파일 2: `src/components/Auth/register.js` (1번째 줄)**
```js
// 수정 전

```

**파일 3: `src/components/board/board.js` (107번째 줄)**
```js
// 수정 전
```

## 3-8. myparty.js — escapeHtml 함수 중복 제거 (중요도: 중간)

### 문제

`src/scripts/myparty.js` 608~612번째 줄에 `escapeHtml` 함수가 정의되어 있는데, `src/utils/escapeHtml.js`에 이미 같은 함수가 있어요.

### 해결 방법

**Step 1.** `myparty.js` 맨 위에 import를 추가하세요.

```js
// src/scripts/myparty.js — 맨 위 import 부분에 추가
import { escapeHtml } from '../utils/escapeHtml.js';
```

**Step 2.** 608~612번째 줄의 `escapeHtml` 함수를 삭제하세요.

```js
// 삭제 (myparty.js 608~612번째 줄)
function escapeHtml(text) {
  const div = document.createElement('div');
  div.textContent = text;
  return div.innerHTML;
}
```

**왜 이렇게 고쳐야 하나요?**

`utils/escapeHtml.js`는 `null`과 `undefined`도 안전하게 처리해요(`String(text ?? "")`). `myparty.js`의 로컬 버전은 `null`이 오면 문자열 `"null"`이 돼요. 같은 기능인데 미묘하게 다르면 예상치 못한 버그가 생겨요.

> **쉽게 말하면:** 집에 가위가 2개 있는데, 하나는 잘 드는 가위고 하나는 좀 무딘 가위예요. 잘 드는 가위 하나만 있으면 되고, 무딘 가위는 치우는 게 좋아요.

## 3-11. my.js — 회원탈퇴 API 호출 추가 (중요도: 중간)

### 문제

`src/components/mypage/my.js` 42~47번째 줄에서 회원탈퇴 확인 후 API를 호출하지 않고 바로 리다이렉트해요.

### 해결 방법

```js
// 수정 전 (my.js 42~47번째 줄)
  btnWithdraw.addEventListener('click', () => {
    if (confirm('정말 탈퇴하시겠습니까? 모든 데이터가 삭제됩니다.')) {
      alert('회원탈퇴가 완료되었습니다.');
      window.location.href = '/';
    }
  });

// 수정 후
  btnWithdraw.addEventListener('click', async () => {
    if (!confirm('정말 탈퇴하시겠습니까? 모든 데이터가 삭제됩니다.')) return;

    try {
      const token = localStorage.getItem('token');
      const res = await fetch(`${BASE_URL}/user/me`, {
        method: 'DELETE',
        headers: { Authorization: `Bearer ${token}` },
      });
      if (!res.ok) throw new Error('회원탈퇴 실패');

      localStorage.removeItem('token');
      localStorage.removeItem('userId');
      alert('회원탈퇴가 완료되었습니다.');
      window.location.href = '/';
    } catch (error) {
      console.error(error);
      alert('회원탈퇴에 실패했습니다. 다시 시도해주세요.');
    }
  });
```

**주의:** `my.js` 파일에서 `BASE_URL`을 사용하려면 import가 필요해요:

```js
// my.js 맨 위에 추가
import { BASE_URL } from '../../api/config.js';
```

또는 `initMyInfo`처럼 `import.meta.env.VITE_BASE_URL`을 직접 사용해도 돼요.

> **쉽게 말하면:** "퇴실하겠습니다"라고 말만 하고 방을 나가면, 호텔 시스템에는 아직 투숙 중이에요. 프론트에서 체크아웃 처리(API 호출)를 해야 진짜 퇴실이 돼요.
