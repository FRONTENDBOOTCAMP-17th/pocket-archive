# 📋 백엔드 API 수정 및 기능 추가 요청사항

### 1. 회원가입(Signup) API 응답 구조 변경

사용자 경험 개선을 위해 회원가입 완료 후 별도의 로그인 절차 없이 즉시 메인 페이지로 진입하고자 합니다. 응답 바디에 **Access Token**을 포함해 주세요.

- **변경 전:** 유저 기본 정보만 반환
- **변경 후:** 유저 정보 + **`accessToken`** 포함
- **응답 예시 (Expected):**

```json
{
  "id": 3,
  "loginId": "newuser@test.com",
  "nickname": "포켓몬마스터",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "createdAt": "2026-04-01T10:00:00"
}
```

### 2. 게시글 작성(Post) API Body 필드 추가

게시글당 1장의 이미지만 사용하는 1:1 구조로 확정되었습니다.
기존의 이미지 업로드 API(`v1/images`)는 그대로 유지하되, 게시글 저장 시 해당 이미지의 URL을 함께 저장할 수 있도록 필드를 추가해 주세요.

- **대상 API:** `POST /api/pocket-archive/v1/posts`
- **변경 사항:** `body` 내에 **`imgUrl` (String)** 필드 추가
- **수정된 요청 바디 (Request Body):**
  ```json
  {
    "title": "string",
    "content": "string",
    "category": "string",
    "preset": "DEFAULT",
    "imgUrl": "https://storage.fullstackfamily.com/.../image.webp"
  }
  ```
- **참고 사항:** 이미지는 사전 업로드 방식을 사용하므로, 프론트엔드에서 `POST /v1/images` 호출 후 받은 URL을 위 필드에 담아 보낼 예정입니다.
  - 이미지가 없는 경우 해당 필드는 `null` 또는 빈 문자열(`""`) 처리가 가능해야 합니다.
